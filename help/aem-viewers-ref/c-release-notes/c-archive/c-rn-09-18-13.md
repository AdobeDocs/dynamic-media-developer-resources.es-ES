# Notas de la versión de visores de Scene7 4.9.2{#scene-viewers-release-notes}

Mejoras, correcciones de errores y problemas conocidos en visores de Scene7 4.9.2

## Mejoras en visores de Scene7 4.9.2 {#section-0b0363fae0274737bace119f281e740f}

Las actualizaciones del visor son retrocompatibles y, por lo tanto, no es necesario realizar cambios en el código web existente. Sin embargo, se recomienda probar los nuevos visores en nuestro entorno de ensayo. Póngase en contacto con el servicio de asistencia técnica para obtener instrucciones sobre cómo acceder al servidor de ensayo para poder probar los visores. Una vez finalizado, puede consultar su sitio web para probar las actualizaciones.

* Se han aumentado los requisitos mínimos para los visores en iOS6.
* Se Añadió la compatibilidad con el seguimiento de eventos personalizado para los visores.
* Compatibilidad Añadida para establecer la velocidad de bits inicial para el visor de vídeo.
* El visor de vídeo ahora utiliza de forma predeterminada el flujo HLS en el escritorio Safari.
* Se han refactorizado las sugerencias para corregir varios errores.
* Se han eliminado las restricciones de funciones sociales en los visores de catálogos electrónicos y vídeos de los dispositivos móviles.

## Correcciones de errores en visores de Scene7 4.9.2 {#section-405e47e202e54943bd843d82415c91cb}

Ahora se han corregido los siguientes errores:

* los visores se mostraban con un tamaño incorrecto después de volver de la pantalla completa e intercambiar recursos.
* los visores mostraban el botón de cierre en modo de pantalla completa.
* El visor de catálogos electrónicos no mostraba mapas de imagen en modo vertical en dispositivos móviles.
* El visor de catálogos electrónicos mostraba botones de panorámica en teléfonos móviles.
* La información de herramientas predeterminada del visor de catálogos electrónicos no se aplicaba si el ID de contenedor no era &quot;ecatalog&quot;.
* La información del visor de catálogos electrónicos se ocultaba detrás de las miniaturas en la vista de cuadrícula.
* El visor de catálogos electrónicos mostraba un divisor de página para una sola página.
* Los mapas de imagen del visor de catálogos electrónicos no funcionaban en IE9.
* El visor de medios mixtos restablecía la posición de la barra de desplazamiento de vídeo después de cambiar el tamaño del visor.
* El visor de medios mixtos utilizaba la ilustración incorrecta para los botones de giro de panorámica.
* El vídeo del visor de medios mixtos mostraba información sobre herramientas en muestras de medios mixtos.
* Los botones de giro del visor de medios mixtos HTMl5 se mostraban en tabletas.
* La información de objeto del visor de giros y zoom se recortaba por arista en el visor incrustado.
* La posición de la información de objeto de uso compartido en redes sociales se ha desplazado para los botones sociales.
* El formato de sugerencias de uso compartido en redes sociales no coincidía con la información del visor.
* La información sobre herramientas no se mostraba en modo de pantalla completa en Mac OS con Safari 5.
* El tamaño del visor de vídeo no es correcto para la barra de progreso cuando se rota el iPad y se cambia entre los modos de pantalla.
* El visor de vídeo generaba registros de consola de forma predeterminada.

## Problemas conocidos en visores de Scene7 4.9.2 {#section-368dd0dc138b44e28b49314e4ad0fa1a}

**Todos los visores de Scene7**

* No se admiten marcas de agua, confusión ni bloqueo.

**Todos los visores**

* Los parámetros con nombres de instancia explícitos en el código deben sobrescribirse; también se deben sobrescribir los nombres de instancia en una dirección URL. Por ejemplo, `zoomView.iconeffect=0`.
* No se admite el recorte de comandos del servicio de imágenes.
* El botón Cerrar solo funciona si el visor está abierto en una ventana secundaria.
* Para personalizar el formato de información de objeto, agregue `!IMPORTANT` a la declaración CSS.

**Visor de catálogos electrónicos**

* No se admiten plantillas JavaScript en mapas de imagen.
* Al desplazarse a otra página HTML y volver ocasionalmente, el visor vuelve a la primera página.
* Establezca `ImageMapEffect` el modificador rollover en `1` para invocar infopanels.

* `Frametransition` definido como `none` o `fade` no admitido.

* El diseño de página se muestra de forma ocasional incorrectamente después de rotar el dispositivo iOS. Al acercar la página se corrige el diseño.

**Visor de medios mixtos**

* En este momento no se admite la reproducción de pistas de sonido.

**Visor social**

* Para procesar las miniaturas correctamente en el correo electrónico saliente, el modificador debe tener una dirección URL absoluta. `serverurl`

**Visor de vídeo**

* La imagen de póster puede encontrar el error &quot;tamaño máximo&quot;. Es posible que el usuario tenga que aumentar la configuración de límite para la publicación con servicio de imágenes.
* Los subtítulos de vídeo requieren un conjunto de reglas de compañía si están alojando una página HTML que se suministra desde un servidor externo que no es un servidor de Scene7. Póngase en contacto con el servicio de asistencia técnica de Adobe para obtener ayuda.

**Visores Flash AS3-all**

* Doble codifica el carácter # en los nombres de los recursos.
* La compatibilidad con el servidor no puede cargar animaciones SWF con vídeos incrustados.
* La compatibilidad con el servidor no puede cargar apariencias de visor si se compila para Flash Player 6. La solución es compilar para Flash Player 7.
* Macintosh OS y Flash Player versión 10,0,32,18: La solución depende de un puente JavaScript en lugar de una conexión local para comunicarse entre Flex y Flash. Por lo tanto, la aplicación Flex debe estar incrustada en el envoltorio HTML.
* Actualmente, los visores Flash admiten archivos SWF que solo se cumplen para Flash 7.
* Debido a un problema con Java v.1.5.0_06, el componente de servidor no es compatible con esa versión de Java. [http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990](http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990).
* Para un parámetro de URL personalizado de apariencia, el parámetro de la URL de apariencia debe tener codificación de dirección URL.
* Los conjuntos de imágenes con caracteres ASCII extendidos en el nombre deben tener codificación URL dos veces cuando se envían al visor o la solicitud FVCTX falla porque Flash Player descodifica el nombre antes de pasarlo al código del visor.
* Si la página HTML de incrustación tiene contenido significativo después del visor, es posible que el visor llame a una función de JavaScript antes de que la página se cargue completamente. Esto puede anular la carga de página. Una solución recomendada es utilizar un controlador de evento onLoad para retrasar la inicialización del visor hasta que se complete la carga de la página.
* Los cambios realizados en el `S7Config.setFlashParam()` permiten que se pase más de un parámetro/valor con esto `syntax S7Config.setFlashParam(<viewer ID>, <parameter name>, <parameter value> [, <parameter name>, <parameter value>])`, pero en ese caso el primer parámetro que se pase debe ser el parámetro &quot;image&quot; seguido del parámetro &quot;currentFrame&quot; (si es necesario) y, a continuación, cualquier otro parámetro.

* Cuando se utiliza el `skipFrames=frameList` parámetro para omitir imágenes, el visor devuelve un error si la función de cambio de imagen de JavaScript llama directamente a un marco omitido `InitialFrame`, o si se trata de un objeto RenderSet que tiene dos o más muestras asociadas a una imagen, cuando se selecciona la combinación de muestras que hace referencia al marco omitido.

* El recorte no se admite junto con destinatarios de zoom o muestras en los argumentos `image=` o `modifier=` .

* El argumento `rgn` no se admite en los visores de flash para el modificador de imagen.
* Un nuevo parche de seguridad IE requiere que el usuario active Flash y otras aplicaciones ActiveX en IE. Consulte el sitio de Microsoft para obtener información sobre soluciones alternativas.
* Los problemas de seguridad impiden que el visor pueda leer desde un servidor diferente en el mismo dominio, a menos que ambas direcciones URL sean nombres de dominio completos o que la &quot;directiva entre dominios&quot; esté configurada específicamente para permitir el acceso desde ese servidor.
* Con las nuevas funciones de seguridad de Flash 7, si los visores intentan leer desde un servidor diferente en el mismo dominio desde donde se encuentran los visores (es decir, se redirigen mediante las rutas de acceso en el `infoServerUrl`, `searchEngineUrl`o `serverUrl parameters`), una alerta de seguridad notifica al usuario este hecho y le pregunta si desea permitir esto, a menos que la &quot;política entre dominios&quot; esté configurada para permitir el acceso desde ese servidor.

   Para obtener más información sobre cómo configurar una &quot;política entre dominios&quot;, consulte el siguiente artículo: [http://kb2.adobe.com/cps/142/tn_14213.html](http://kb2.adobe.com/cps/142/tn_14213.html).

* Error de excepción generado cuando el visor está incrustado en la página HTTPS. El problema se debe a la comunicación con el registro del servidor. Para obtener más información: [http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html](http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html)
