## Elementos destacados de la versión {#section-524cffc79c1f4bb38627a5ecccd0bf52}

* Se ha agregado compatibilidad con el diseño web interactivo: servicio de imágenes interactivas basadas en puntos de interrupción y visores HTML5 interactivos (conjuntos de imágenes, vídeo, medios mixtos, catálogo electrónico, conjuntos de giros, conjuntos de muestras, zoom) que cambian de tamaño dinámicamente según el diseño de la página.
* Integración de informes de vídeo de Adobe Analytics.
* Personalización avanzada del visor HTML5: cree interacciones personalizadas del visor con la página web de incrustación.
* Mejora de imagen: se ha agregado la capacidad de recortar imágenes a una ruta incrustada durante la entrega.

## Visores {#section-609885a9c8514a4095da74a0b526138d}

Se actualizaron los siguientes elementos:

* El requisito mínimo del sistema para el sistema operativo Windows es ahora Windows 7.
* El requisito mínimo del sistema para Internet Explorer es ahora la versión 9.
* El requisito mínimo del sistema para Mac OS ahora es Mac OS 10.8.
* El requisito mínimo del sistema para Safari ahora es la versión 6.

Se han refactorizado los siguientes elementos:

* Los visores HTML5 admiten el diseño interactivo.
* Los visores HTML5 utilizan entradas CSS y marcadores de tamaño para controlar la visualización en equipos de escritorio y dispositivos móviles.

Ahora se admite lo siguiente:

* Integración de los informes de vídeo de Adobe Analytics para el visor de vídeo HTML5 y el visor de medios mixtos.
* Seguimiento de eventos de varios visores HTML5 en la misma página.
* Se ha agregado `SetIndicator` al visor de zoom HTML5 y al visor de medios mixtos HTML5 para representar muestras visualmente en dispositivos móviles.
* Se ha agregado la API `getComponent` para permitir el acceso externo a los componentes del visor HTML5.
* Los parámetros de configuración del visor HTML5 se pasan como un único objeto JSON.
* Los subtítulos WebVTT se pasarán como objeto JSON, lo que elimina la necesidad de configurar el conjunto de reglas.
* Plantillas JavaScript en el visor de catálogos electrónicos HTML5 para vincular mapas de imágenes a páginas externas.
* Vinculación a páginas dentro del visor de catálogos electrónicos HTML5.
* `stagesize` al visor flotante HTML5.
* `initComplete` a los visores HTML5 para activar funciones después de los visores.
* Controlador de Analytics para visores HTML5 para admitir el seguimiento de terceros.
* Se ha agregado el botón &quot;Seleccionar todo&quot; para que sea más fácil obtener vínculos y código incrustado para el uso compartido en redes sociales.
* Se ha añadido un cambio de tamaño para la vista de zoom flotante.
* Configuración de puntos de interrupción de recarga de imagen para la vista de zoom flotante cuando se utiliza con un diseño interactivo.
* Se agregó compatibilidad con clics para ajustar el volumen.
* SpinView y ZoomView para aplicar gestos a la página web de incrustación cuando no interactúe con el visor.
* Se ha agregado la API `unload()` a SpinView y ZoomView para permitir la eliminación del componente durante la ejecución.
* El visualizador de vídeo aplicará gestos a la página web de incrustación cuando no interactúe con el visualizador.
* Configuración de la región de zoom inicial para ZoomView.
* ZoomView para ocultar el componente `IconEffect` cuando el zoom no es posible

Consulte también [Guía de referencia de visores](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/library/home.html).

## Imágenes {#section-113f27ea04cd4998a324bba71519c93c}

* Compatibilidad con JPEG progresivo.
* Se ha cambiado la configuración predeterminada de máscara de enfoque para nuevos clientes.
* Se agregaron operaciones de máscara de enfoque a trabajos de recursos de reprocesamiento.
* Se ha añadido una advertencia al eliminar ajustes preestablecidos de imagen.
* Configuración de puntos de interrupción de recarga de imagen cuando se utiliza para un diseño interactivo.

## Plataforma {#section-7e2e7bd11b81424e8cd939b6675ca729}

* Los archivos entrantes ahora se comprueban en busca de virus usando Clam AntiVirus antes de la ingesta. Los archivos con errores se ponen en cuarentena inmediatamente.
* Amplíe la ventana emergente de detalles del trabajo para incluir la hora de cada registro de detalle.
* Se ha implementado la API para eliminar un valor de metadatos de un recurso (deleteAssetMetadata).
* Se ha cambiado el comportamiento de búsqueda predeterminado para incluir dc.subject.
* Se ha cambiado el mecanismo de restablecimiento de contraseña (Contraseña olvidada) a un proceso de varios pasos para evitar DOS.
* Convertir el correo electrónico de contraseña olvidada de HTML a texto sin formato para evitar la inclusión en la lista de bloqueados.
* Se ha eliminado la función predeterminada.
* Además de la capacidad de cargar contenido en Scene7 Publishing System mediante la carga de directorios de servidor (ftp) y la carga de URL, ahora también puede cargar contenido mediante el POST HTTP. Consulte la [API del sistema de producción de imágenes](/help/aem-ips-api/c-overview.md) para obtener más información sobre el uso del POST HTTP.

## Correcciones de errores {#section-456515477c8a4aa69309de43292ce5ed}

**Visores**

* Posibles problemas de seguridad de los visores flotantes y de zoom rápido en Flash.
* Los visores HTML5 no se cargan debido a un agente de usuario no reconocido.
* La función de arrastrar y soltar del visor de catálogos electrónicos HTML5 no funciona en Internet Explorer con la página que contiene mapas de imágenes.
* El visor de vídeo HTML5 evita el acceso para desbloquear el complemento de Flash.
* Los visores HTML5 evitan la impresión en el navegador Firefox.
* El visor de zoom HTML5 muestra una ligera fluctuación al iniciar o restablecer el zoom.
* Compatibilidad HTTPS con visores HTML5 y AS3.
* autoResize no debe estar en el visor flotante HTML Copiar URL/Código incrustado.
* No hay modificador videoserverurl en la URL de copia del visor de vídeo HTML5.
* El parámetro `contenturl` procede de la URL de vista previa del visor HTML5 del servidor de publicación.

**Imágenes**

* No se puede agregar la vista ajustada como imagen predeterminada en la configuración de Publicación de imágenes.
* Cambie la etiqueta colorManagementOptions &quot;Convertir en SRGB (predeterminado)&quot; por &quot;Gestión de color predeterminada&quot;.
* Cuando se encuentra en Archivo > Editar información > seleccione varios recursos de imagen, la información de edición no estaba disponible.

**Plataforma**

* Se agregó un mensaje de alerta cuando edita la configuración de Adobe Analytics y luego intenta escapar de la configuración sin guardar.
* La referencia de opciones del proyecto se restablece en Ninguno al editar un trabajo de carga programado.
* La carpeta File-Move no debe mostrarse para el rol Usuario.
* No se podían ver algunas de las pestañas en la vista Detalles.
* Error de edición de impresión virtual cuando se instala la versión de depuración de Flash Player.
* &quot;Espacio después&quot; no funciona; debería usar &#39;sa&#39;, actualmente es &#39;margb&#39;.
* Al inicializar la configuración de la empresa, Scene7 Publishing System envía la opción `<defaultValue>` duplicada en la API `<createMetadataField>`.

* El complemento de Flash y Scene7 Publishing System se bloquean debido a un gran número de carpetas que no tienen suficiente memoria de Flash.
* Al quitar el valor del campo definido por el usuario Fecha/Hora, Scene7 Publishing System no envía una solicitud a IPS (Image Publishing System).
* Si las opciones de codificación de Evideo están habilitadas, el reprocesamiento de un archivo PDF o PSD ha encontrado el mensaje de advertencia Codificación de vídeo adaptable .
* Error al exportar recursos cuando SSL está habilitado.
* Se mostraba una contraseña incorrecta en el correo electrónico de contraseña olvidada.
* Cualquier solicitud de administrador web de Scene7 Publishing System se redirige a HTTP.
* Todos los ajustes preestablecidos de conjunto de lotes se aplicaban durante la carga si no se seleccionaban ajustes preestablecidos.
* Scene7 Publishing System solicita s7info mediante HTTP al cargar contenido de s7info.
* La dirección URL del correo electrónico de contraseña olvidada es HTTP, incluso si procede de SSL Scene7 Publishing System.
* Los caracteres especiales se codificaban en un correo electrónico &quot;Olvidó la contraseña&quot; con formato HTML, por lo que la contraseña no se mostraba correctamente.
* Falta la información de la cuenta de FTP de Scene7 en el correo electrónico de bienvenida de una cuenta de prueba
* Error en el trabajo de reprocesamiento cuando se seleccionan las opciones de proyecto y de recorte.
* El archivo de carga no funciona cuando las opciones de trabajo se restablecen a los valores predeterminados de la empresa.
* La etiqueta Empresa no se muestra completamente en la página Configuración de Adobe Analytics.
* No se pueden importar metadatos.

## Problemas conocidos {#section-b2608177928b4174b080d89c938bb0d3}

* Se espera que la actualización de mayo de 2014 de Adobe Illustrator CC (Creative Cloud) y Adobe InDesign CC introduzca incompatibilidades con los complementos que crean FXG de Scene7. Estas incompatibilidades afectan a los usuarios de la solución de impresión virtual de Scene7. Si utiliza la impresión virtual de Scene7, no actualice Illustrator y el InDesign a esta actualización del Creative Cloud. En su lugar, siga usando la versión anterior a mayo de 2014. Adobe Systems espera restaurar la compatibilidad de la creación de FXG de Scene7 con versiones más recientes de Illustrator y InDesign en otoño de 2014.
* El botón Incrustar código no funciona cuando el ajuste preestablecido Visor universal contiene el ajuste preestablecido AS2 predeterminado desactivado.
* El reprocesamiento del conjunto de vídeos adaptables siempre crea un nuevo conjunto de vídeos adaptables.
* La interfaz de usuario del cuadro de diálogo Detalles del trabajo está truncada a la derecha después de añadir la hora en los detalles.

## Funciones que ya no son compatibles con {#section-7b04b6fdb8d1473880ccaf84211254ad}

* La versión de otoño de Scene7 6.9 no admitirá la división mediante guiones de texto. Los diccionarios de división mediante guiones definen cómo se pueden dividir y dividir las palabras para admitir un flujo de texto óptimo.
* A partir del 31 de enero de 2014, Scene7 dejó oficialmente de ofrecer asistencia para la plataforma del visor DHTML.

   Para obtener más información sobre este cambio, consulte el siguiente sitio web de preguntas frecuentes:

   [http://dev.day.com/content/docs/en/cq/current/administering/integrating_with_omniture/scene7/dhtml-viewer-endoflifefaqs.html](http://dev.day.com/content/docs/en/cq/current/administering/integrating_with_omniture/scene7/dhtml-viewer-endoflifefaqs.html)

* En esta versión ya no se utilizan letreros dinámicos (Flash). La adopción de esta función ha sido baja, ya que han surgido otras tecnologías para crear, administrar y entregar banners personalizados de sitios web. La funcionalidad de banner dinámico (Flash) de Scene7 seguirá funcionando en el servidor, pero ya no será compatible.
* Recordatorio: Desde la versión anterior de Scene7, se ha dejado de admitir oficialmente el servidor de metadatos y el uso de las API asociadas.
* Las extensiones de Creative Suite de Scene7 (paneles de Flash) quedarán obsoletas a finales de 2014. El desarrollo de los flujos de trabajo de ingesta de contenido se centra en Adobe Experience Manager Dynamic Media y Creative Cloud. Para Scene7, las funciones de carga y exportación de Scene7 Publishing System permanecerán y se podrán utilizar en su lugar.
* A partir de mayo de 2014, Akamai ya no es compatible con la notificación por correo electrónico cuando se completa la invalidación de caché en Scene7. El tiempo de procesamiento típico para la invalidación de CDN es de aproximadamente ocho minutos. Si hay una gran cola de solicitudes o un gran número de direcciones URL, el tiempo de procesamiento puede tardar un poco más.
* Categoría &quot;Ajustes preestablecidos universales&quot; en la dirección URL y en la ficha Código incrustado de la vista de detalles.
* Flujo RTMP para la vista de detalles de vídeo.
* &quot;Comunicaciones por correo electrónico&quot; y servlet SendMailSimple.jsp desde Scene7 Publishing System.
* Sección de cuenta de FTP de Scene7 en la plantilla de correo electrónico de bienvenida para el usuario de Media Portal.
* dc.title desde el panel de filtro, ya que hacía que la instancia de IPS (Image Publishing System) no funcionara.
* Las funciones sociales se muestran en modo de pantalla completa en los visores HTML5.
* El botón Cerrar se muestra en modo de pantalla completa en los visores HTML5.

## Nuevas funciones y correcciones de errores en Scene7 SDK 2.7.1 {#section-b82b4f546ce44a769e94fb1493511035}

**General**

* Se han aumentado los requisitos mínimos para iOS6.
* Corregido: La información sobre herramientas de Safari se ha desplazado de la pantalla al modo de pantalla completa.
* Se han refactorizado las informaciones de objeto para corregir varios errores.

**Botón**

* Se ha agregado compatibilidad con CSS en los estilos de botón predeterminados.

**Contenedor**

* Corregido: Tamaño incorrecto del contenedor después de volver de la pantalla completa e intercambiar recursos.
* Corregido: El contenedor coordina incorrectamente si el contenedor se desplaza mediante una transformación CSS.

**Uso compartido de correo electrónico**

* Corregido: Objeto de información de objeto que aparece con el botón Quitar.
* Corregido: La información del objeto de la barra de desplazamiento se muestra detrás de la interfaz de usuario y se recorta.
* Corregido: La información del objeto &quot;Cancelar&quot; aparece detrás del botón Enviar correo electrónico.

**Uso compartido incrustado**

* Corregido: El texto de código incrustado no se ajusta y aparece sobre la barra de desplazamiento de Internet Explorer.

**Vista de zoom flotante**

* Corregido: Los eventos flotantes no se envían en Internet Explorer 8.

**Volumen silenciable**

* Corregido: La información del objeto de volumen se muestra debajo de la barra de volumen.

**Muestras**

* Corregido: El método `selectSwatch` no se activa cuando está incrustado en la función `initViewer()`.

**Uso compartido social**

* Se agregó compatibilidad con el modificador de cojinetes para controlar la ubicación de diseño de los botones de uso compartido.
* Se han agregado eventos para controlar la pausa y reproducción al interactuar con funciones sociales.
* Corregido: El icono Uso compartido en medios sociales no se desvanece con el `controlbar` en Zoom.
* Corregido: El icono de uso compartido social no aparece al fundirse inicialmente con Galaxy SIII.
* Corregido: Los cuadros de diálogo modales de uso compartido en medios sociales no se muestran en el modo de pantalla completa del iPad3.
* Corregido: Los cuadros de diálogo modales de uso compartido en medios sociales no se muestran en el modo de pantalla completa en Mac con Safari 5.1.
* Corregido: Los cuadros de diálogo modales de uso compartido social aparecen fuera de la pantalla después del desplazamiento.

**Vista de cuadrícula en miniatura**

* Corregido: ThumbnailGridView no envía `AssetEvent.ASSET_CHANGED`.

**Reproductor de videos**

* Se agregó compatibilidad con el modificador `initialBitRate` para seleccionar la velocidad de bits utilizada para la reproducción inicial en el escritorio.
* Se ha implementado el proxy de vídeo para iPhone para solucionar problemas de vídeo en dispositivos iPhone.
* Se agregó compatibilidad con el detector de eventos para pausar la reproducción cuando se invocan características sociales.
* El modo de reproducción establecido en auto ahora utiliza un reproductor nativo en el escritorio de Safari.
* Corregido: Protocolo HTTPS que genera una advertencia de seguridad en IE8 usando la reproducción de Flash.
* Corregido: El tamaño del vídeo no es correcto en el modo de pantalla completa de Safari 5 con la reproducción en Flash.
* Corregido: Las muestras del conjunto de medios no se pueden seleccionar después de que el vídeo se reproduzca en iPhone.

**Vista de zoom**

* Corregido: Artefactos mostrados después de acercar la imagen a una imagen o girar el dispositivo en Android.

## Nuevas funciones y correcciones de errores en S7OnDemand 5.0.1 {#section-ba3a595134144711bf27dd8dca249e5d}

* Corregido: Compatibilidad HTTPS con visores HTML5 y AS3.
* Se ha agregado compatibilidad interactiva con el código incrustado de visores HTML5.
* El código incrustado para los visores HTML5 pasa los parámetros de configuración como un solo objeto JSON.
