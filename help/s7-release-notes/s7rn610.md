## Scene7 IPS (Image Production System) 4.5.0 {#section-3e37ed37b6d146419f4578bd7ffeaa74}

**Nuevas funciones**

* Ahora se ejecuta a través de JDK 1.8.
* Compatibilidad con la etiqueta de orientación EXIF.
* Cargue archivos de directorio para archivos de FTP temporales (como se crea con ProFTP).
* La publicación instantánea ahora es compatible con la representación de imágenes. Esto significa que los recursos de viñeta procesados están disponibles inmediatamente para su lanzamiento en el sitio web, sin necesidad de iniciar un trabajo de publicación. Si está interesado en utilizar esta función, póngase en contacto con el soporte técnico para que habiliten su cuenta para la publicación instantánea.

**Funciones que ya no se admiten**

* Opción de resincronización del servidor de catálogo dentro de la opción de validación de publicación de IPS Classic - eliminada.
* Ya no se pueden definir valores de campo de etiqueta de metadatos vacíos o duplicados.

**Correcciones de errores**

* El `<metadataArray>` estaba vacío para el archivo [!DNL .vtt] como respuesta a la API `getAssets`.

* Falta el campo de datos de usuario para la orientación.
* La respuesta de `getJobLogDetails` para un trabajo que actualizaba un archivo ACO no especificaba correctamente los detalles del trabajo.
* El archivo principal PSD se registró como publicado varias veces por servidor de imágenes.
* Los recursos con caracteres no válidos en su nombre se pueden cargar en IPS.
* Las miniaturas de vídeo que no se generaban se dejaban en estado &quot;Pendiente en cola&quot;.
* Condición de carrera con restauración de carpetas entre la carga de directorio de servidor y `deleteFolder` grande.
* Los archivos PDF se registraban como publicados dos veces por servidor de publicación cuando se publicaban por anticipado en la carga.
* La carga de varios archivos PSD con los mismos nombres de capas provocó errores de optimización.
* La rasterización PDFL del PDF del cliente daba como resultado una línea blanca inesperada en la imagen generada.
* Los archivos de miniaturas de vídeo se publicaron varias veces al cargarlos.
* La detección de archivos de carga duplicados generó dos notificaciones de carga, lo que anuló la contabilidad de archivos completada.
* El archivo XML de Sidecar para la transcodificación de vídeo se registró como publicado varias veces al actualizarse.
* moveFolder falló al eliminar la carpeta anidada y mover otra carpeta (que no está en la carpeta principal de la carpeta anidada y tiene el mismo nombre que la carpeta anidada) a la ubicación donde solía estar la carpeta anidada.
* Las entradas de configuración de estado de clúster en la configuración local y compartida no se purgaron mediante la limpieza.
* Cambie el analizador de registros de API para filtrar por formatos de direcciones de correo electrónico.
* El procesamiento de vídeo no se asigna `Iptc4xmpCore:Location` correctamente.
* Existía una condición de carrera entre la limpieza y la carga.
* La publicación no utiliza `assetFile`, por lo que la publicación provisional solo funciona para recursos nuevos.
* Se ha generado un puntero nulo al actualizar un conjunto de reglas.

## Visores {#section-833ab92c91c941d2bfdc27f233f582ad}

**Nuevas funciones, mejoras y correcciones de errores para los visores de servicio de imágenes de Dynamic Media 5.2.2**

Las actualizaciones del visor son compatibles retrospectivamente: no es necesario realizar ningún cambio en el código web existente. Sin embargo, se recomienda probar los nuevos visores en nuestro entorno de ensayo. El siguiente sitio web le da instrucciones sobre cómo configurar su sistema para acceder a nuestro servidor de ensayo:

[http://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html](http://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html)

Una vez configurado el equipo para acceder al servidor de ensayo, puede comprobar el sitio web para probar la actualización. Si utiliza visores HTML5 predeterminados, la práctica recomendada es que los pruebe con nuestro servidor de ensayo estándar y s7is5-preview-staging.scene7.com.

Nuevas funciones y mejoras

Consulte también [Guía de referencia de visores](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/library/home.html).

* Se agregó compatibilidad con Imprimir, Descargar y Favoritos en el visor de catálogos electrónicos.
* Se ha añadido la capacidad de recuperar ParameterManager mediante la API getComponent.
* Visualizador de giros convertido, visor de zoom, visualizador de vídeo y visor flotante para usar sprites en las ilustraciones.
* Se ha agregado compatibilidad con la pantalla completa nativa de IE11.
* Se ha refactorizado la compatibilidad con pantalla completa simulada (no nativa) en el contenedor.
* Se ha aumentado el tamaño del marcador pequeño CSS para que admita teléfonos más grandes.
* Se han eliminado los marcadores de tamaño CSS para los visores de medios mixtos, de giro HTML5, BasicZoom, Zoom, Giro y de medios mixtos en los navegadores de escritorio.
* Se agregó compatibilidad para permitir la configuración de calidad de fotogramas precargados en SpinView.
* Corregido: Galaxy S4, modo vertical: Se usó un marcador de tamaño CSS incorrecto cuando se habilitó el modo de pantalla completa.
* Corregido: los [mapas de imagen de Catálogo electrónico] se atascan después de usar plantillas de javascript para la vinculación externa en Internet Explorer 9.
* Corregido: Internet Explorer 9 e Internet Explorer 10: zoom funciona incorrectamente si la propiedad width está definida para IMG en CSS.

**Nuevas funciones, mejoras y correcciones de errores para el SDK 2.9.2 del visor HTML5 de Scene7**

General

* Se ha corregido un problema por el que s7sdk.browser.device.version detectaba el quinto Android de forma incorrecta.
* Se ha eliminado el uso de visibilidad:estilo visible en los componentes del SDK.

Vista de zoom flotante

* Corregido - [IE9] El Flotante a veces permanecía en la pantalla después de mover el ratón.
* Corregido: la cadena de iscommand se agregaba dos veces a la imagen inicial.
* Se agregaron eventos de estado adicionales.
